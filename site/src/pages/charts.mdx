have made them generally available for others to use. You may be able to find some of these charts elsewhere, but many

```shell
helm repo add mya https://mya.sh
```

## Provenance and Integrity

I recently introduced GPG signing to my helm charts. If you'd like to verify signatures before deployment, you can
receive my GPG public key from [Ubuntu's Keyserver][] using the following command:

```shell
gpg --recv-keys 0x0fca5e3c09dfb314
```

Once you've received my GPG key, you can verify charts during install using the `helm install --verify`.

[Ubuntu's Keyserver]: https://keyserver.ubuntu.com/pks/lookup?op=get&search=0xfd0f1e353074e15a4883bc630fca5e3c09dfb314

## Contents

[`12factor`](https://github.com/mjpitz/mjpitz/tree/main/charts/12factor)<br/>
Easily deploy an application that conforms to 12factor application patterns.

[`cognative`](https://github.com/mjpitz/mjpitz/tree/main/charts/cognative)<br/>
Deploy the [Cognative][] business intelligence and operations platform.

[`drone`](https://github.com/mjpitz/mjpitz/tree/main/charts/drone)<br/>
Deploys a [Drone][] server and optionally configure a `digitalocean` or `kubernetes` runner.

[`gitea`](https://github.com/mjpitz/mjpitz/tree/main/charts/gitea)<br/>
Git with a cup of tea. [Gitea][] is a painless, self-hosted Git Service.

[`litestream`](https://github.com/mjpitz/mjpitz/tree/main/charts/litestream)<br/>
Easily add a [Litestream][] sidecar to any `Deployment`, `DaemonSet`, or `StatefulSet`.

[`maddy`](https://github.com/mjpitz/mjpitz/tree/main/charts/maddy)<br/>
Deploys a single-tenant Maddy email server.

[`raw`](https://github.com/mjpitz/mjpitz/tree/main/charts/raw)<br/>
Deploys raw Kubernetes resources to the cluster.

[`redis`](https://github.com/mjpitz/mjpitz/tree/main/charts/redis)<br/>
Deploys an inconsistent Redis cluster, intended to be fronted by an [Envoy][] sidecar.

[`redis-queue`](https://github.com/mjpitz/mjpitz/tree/main/charts/redis-queue)<br/>
Deploy a single durable Redis instance, intended to be used as a work queue.

[`redis-raft`](https://github.com/mjpitz/mjpitz/tree/main/charts/redis-raft)<br/>
Deploy a consistent, partition-tolerant Redis cluster backed by the Raft consensus protocol.

[`registry`](https://github.com/mjpitz/mjpitz/tree/main/charts/registry)<br/>
Deploys a replicated container registry, with optional caching provided by `redis`.

[`renovate`](https://github.com/mjpitz/mjpitz/tree/main/charts/renovate)<br/>
Deploys Renovate for automatically managing your software dependencies.

[`storj`](https://github.com/mjpitz/mjpitz/tree/main/charts/storj)<br/>
Easily add a Storj S3 Gateway as a sidecar to any `Deployment`, `DaemonSet`, or `StatefulSet`.
